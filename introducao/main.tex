\section{Programação Competitiva}

Seja muito bem-vindo ao GEPC! Você está prestes a adentrar em uma longa e árdua jornada, mas que será muito recompensadora. A programação competitiva envolve muitas horas de estudo sobre lógica e resolução de problemas, e iremos auxiliar você durante todo o processo. Sem mais delongas, vamos introduzi-lo(a) às competições mais famosas do Brasil.

\subsection{Olimpíada Brasileira de Informática (OBI)}
A OBI é a maior competição individual do gênero no Brasil. Ela se divide em Iniciação (lógica teórica) e Programação (onde o código entra em cena), com níveis que seguem sua trajetória acadêmica, como \textbf{Nível Júnior}, para alunos de Ensino Fundamental, \textbf{Níveis 1 e 2} para Ensino Médio, e \textbf{Nível Sênior}, para estudantes do primeiro ano de graduação. Os melhores de cada nível são convidados para cursos na UNICAMP e os mais jovens podem chegar à Olimpíada Internacional (IOI)

\subsection{Maratona SBC de Programação}
A Maratona da Sociedade Brasileira de Computação (SBC) é a etapa regional do ICPC (International Collegiate Programming Contest). Aqui, o trabalho é em equipe: três pessoas, um computador e muitos balões (cada problema resolvido rende um balão colorido). É um ambiente de alta pressão, muita cafeína e uma satisfação indescritível ao ver o veredito "Accepted" brilhar na tela.

\section{Introdução à Programação}
Programar é, essencialmente, dar ordens detalhadas a um processador que não possui bom senso. Ele fará exatamente o que você pedir, mesmo que o que você pediu seja uma estupidez que trave o sistema.

\subsection{Linguagens de Programação}
Computadores operam apenas com pulsos elétricos (zeros e uns). Como não somos máquinas, usamos as linguagens de programação como uma ponte: elas são um conjunto de regras e palavras que nos permitem descrever passos lógicos de forma humana. Um software chamado compilador lê o que escrevemos e traduz para a linguagem que o hardware entende.\\
No nosso caso, utilizaremos o C++. Ela é a linguagem padrão nas competições por ser extremamente rápida e oferecer a STL (Standard Template Library), uma coleção de ferramentas prontas que evitam que você precise reinventar a roda durante uma prova.

\subsection{IDEs}
A IDE é o ambiente por meio do qual iremos construir nosso código. A diferença entre uma IDE e um editor de texto simples é que a IDE possui ferramentas para auxílio no desenvolvimento e no teste de código. Recomendamos o VS Code ou o Code::Blocks. Lembre-se: em uma competição real, você não terá acesso a extensões luxuosas de preenchimento automático, então tente não se tornar dependente delas.

\section{Primeiros códigos}

\subsection{A função main()}
Todo programa em C++ começa por um lugar específico: a função main. Pense nela como o portão de entrada. O computador ignora tudo o que está fora dela até que ela seja chamada.

\begin{lstlisting}[language=C++]
int main() {
    // Seu código começa aqui
    return 0; 
}
\end{lstlisting}

\textbf{OBS$_1$}: O return 0; ao final indica ao sistema operacional que o programa terminou com sucesso. Se ele retornar qualquer outra coisa, o sistema pode achar que algo deu terrivelmente errado (o que, sejamos honestos, acontecerá bastante no início).\\
\textbf{OBS$_2$}: A linha que contém "//" no início é uma linha de \textbf{Comentário}. Comentários são partes do código que são ignoradas pelo computador, servindo apenas para melhor compreensão do programador.



\section{Variáveis}
O computador possui memória, e as variáveis são nomes que damos a pequenos espaços nessa memória para guardar informações. Em C++, precisamos dizer ao computador exatamente o tipo de dado que vamos guardar.\\
os tipos básicos de variáveis são:
\begin{enumerate}
    \item \textbf{int}: Números inteiros (ex: 10, -500, 42).
    \item \textbf{double}: Números com casas decimais (ex: 3.1415, 0.001).
    \item \textbf{char}: Um único caractere (ex: 'A', 'b', '@').
    \item \textbf{string}: Um conjunto de caracteres, ou seja, texto.
    \item \textbf{bool}: Valores lógicos: true (verdadeiro) ou false (falso).
\end{enumerate}

Segue um exemplo de declaração de variáveis:
 \\
\begin{lstlisting}[language=C++]
int main() {
    int idade = 18;
    double preco = 12.50;
    char inicial = 'G';
    return 0; 
}
\end{lstlisting}

\section{Entrada e Saída de Dados}
Um programa competitivo geralmente lê dados de um arquivo ou do teclado (\textbf{entrada}) e exibe o resultado na tela (\textbf{saída}). No C++, usamos os objetos cin e cout (dentro da biblioteca iostream).

\begin{enumerate}
    \item \textbf{cin >>}: Pega o que o usuário digitou e guarda em uma variável.
    \item \textbf{cout <<}: Exibe o conteúdo de uma variável ou um texto na tela.
    \item \textbf{endl} ou \textbf{\text{\textbackslash n}}: Servem para pular uma linha. 
\end{enumerate}

\begin{lstlisting}[language=C++]
#include <iostream>
using namespace std;

int main() {
    int x;
    cin >> x; // Lê um número
    cout << "O valor digitado foi: " << x << "\n"; // Mostra o número
    return 0; 
}
\end{lstlisting}

\section{Bibliotecas}
Bibliotecas são arquivos que contêm funções prontas. Se quisermos usar o cin e o cout, precisamos avisar ao compilador que usaremos a biblioteca de entrada e saída.\\
A linha \#include <iostream> no topo do código faz exatamente isso.\\
Outros exemplos de bibliotecas são a \textbf{cmath}, que oferece funções matemáticas como raiz quadrada (sqrt) e potências (pow), \textbf{algorithm}, que contém funções poderosas como o sort (que ordena uma lista de números instantaneamente), além das bibliotecas que implementam as chamadas \textbf{Estruturas de Dados}, que iremos aprender posteriormente. \\
\textbf{Nota sobre using namespace std;:} Você verá frequentemente essa linha nos códigos. Ela serve para não precisarmos escrever std::cin ou std::cout toda vez. É um atalho preguiçoso, mas aceitável no contexto de competição.

\section{Um código de exemplo}
Juntando tudo o que vimos, aqui está a estrutura de um programa que lê dois números e imprime a soma deles:

\begin{lstlisting}[language=C++]
#include <iostream>

using namespace std;

int main() {
    // Declaração das variáveis
    int a, b;

    // Entrada de dados
    cin >> a >> b;
    
    // Cálculo e Saída
    int soma = a + b;
    cout << soma << "\n";

    return 0;
}
\end{lstlisting}

\section{Estruturas Condicionais}
Nem sempre o fluxo de instruções do nosso algoritmo serão contínuas. Às vezes, queremos que o computador realize determinada tarefa somente caso uma condição seja atendida. Por exemplo, em um sistema de banco, o usuário não pode sacar dinheiro caso sua conta não possua o valor solicitado. Para fazer esse tipo de fluxo, utilizaremos as \textbf{Estruturas Condicionais}.
\subsection{Operadores Lógicos}
Antes de explorarmos as estruturas condicionais propriamente ditas, precisamos analisar os operadores lógicos. \\
Se lembrarmos dos operadores aritméticos, como soma e multiplicação, veremos que as operações resultam em valores numéricos, do mesmo tipo que os operandos, por exemplo, em: 
\begin{lstlisting}[language=C++]
int x = 10;
int y = 20;
int res = x + y;
\end{lstlisting}
O resultado, \textit{res}, será uma variável do tipo inteiro. \\
Os operadores lógicos, por outro lado, são operadores que nos retornam valores \textit{lógicos}, ou \textit{booleanos}, como você preferir chamar. Por exemplo, o código:

\begin{lstlisting}[language=C++]
bool a = true;
bool b = false;
bool res = x && y;

\end{lstlisting}
Opera com variáveis booleanas. Vamos explorar todos os operadores:
\begin{enumerate}
    \item \textbf{AND (\&\&)}: Resulta em \textbf{true} caso ambos os valores sejam true, resulta em \textbf{false} caso contrário. e.g.:
    \begin{lstlisting}[language=C++]
        bool res;
        bool v1 = true;
        bool f1 = false;
        bool v2 = true;
        
        res = v1 && v2; // res é true
        res = v1 && f1; // res é false
    \end{lstlisting}
    \item \textbf{OR (||)}: Retorna \textbf{true} caso \textbf{pelo menos um} dos valores seja true, retorna \textbf{false} caso contrário. e.g.:
    \begin{lstlisting}[language=C++]
        bool res;
        bool v1 = true;
        bool f1 = false;
        bool v2 = true;
        bool f2 = false;
        
        res = v1 || f1; // res é true
        res = v1 || v2; // res é true
        res = f1 || f2; // res é false
    \end{lstlisting}
    \item \textbf{NOT (!)}: Opera sobre um único valor. Inverte seu valor lógico. e.g.:
    \begin{lstlisting}[language=C++]
        bool res;
        bool v1 = true;
        bool f1 = false;

        res = !v1; // res é false
        res = !f1; // res é true
    \end{lstlisting}
\end{enumerate}
Além destes, em que operamos apenas sobre valores lógicos, podemos também comparar valores numéricos, usando os operadores lógicos de comparação:
\begin{itemize}
    \item \textbf{EQUALS (==)}: Compara dois valores, caso sejam iguais, retorna \textit{true}. Caso contrário, retorna \textit{false}. e.g.:
    \begin{lstlisting}[language=C++]
        bool res;
        double valor1 = 10.3;
        double valor2 = 12.7;
        double valorIgualAo1 = 10.3;

        res = valor1 == valor2 // res é false
        res = valor1 == valorIgualAo1; // res é true
    \end{lstlisting}
    \textbf{OBS: Não confundir "=" (operador de ATRIBUIÇÃO) com "==" (operador de COMPARAÇÃO)}
    \item \textbf{GREATER THAN (>) e GREATER-EQUAL (>=)}: No caso do \textit{greater than}, resulta em \textit{true} se o primeiro valor for \textit{maior} que o segundo. Para o \textit{greater-equal}, retorna \textit{true} caso o se o primeiro valor for \textit{maior ou igual} ao segundo. e.g.:
    \begin{lstlisting}[language=C++]
        bool res;
        double valor1 = 10.3;
        double valor2 = 12.7;
        double valorIgualAo1 = 10.3;

        res = valor1 > valor2 // res é false
        res = valor2 > valor1; // res é true
        res = valor1 > valorIgualAo1; // res é false
        res = valor1 >= valorIgualAo1; // res é true
    \end{lstlisting}
    \item \textbf{LOWER THAN (<) e LOWER-EQUAL (<=)}: Operadores duais de greater-than e greater-equal. Retornam \textit{true} se o primeiro valor for menor que o segundo e menor ou igual, respectivamente. e.g.:
    \begin{lstlisting}[language=C++]
        bool res;
        double valor1 = 10.3;
        double valor2 = 12.7;
        double valorIgualAo1 = 10.3;

        res = valor1 < valor2 // res é true
        res = valor2 < valor1; // res é false
        res = valor1 < valorIgualAo1; // res é false
        res = valor1 <= valorIgualAo1; // res é true
    \end{lstlisting}
\end{itemize}
